# Day 4 授课节奏安排

这份文件给老师自己看，不放进学生教案内容里。

> **说明：** 本文件是老师侧时长安排，学生主看 `day4_6h_0*.ipynb`。
> 教师口播/答案/可抄代码等仅保存在本地 `教师资料/`（不进公开学生仓库）。

## 正确主线顺序（非常重要）

```
模块 0 复习与环境自检
   ↓
模块 1 评估指标（R² / MAE / RMSE）
   ↓
模块 2 多种算法对比
   ↓
模块 3 参数 vs 超参数
   ↓
模块 4 交叉验证与网格搜索
   ↓
模块 5 残差分析
   ↓
模块 6 怎么选模型 + 收束（接 Day 5 经营诊断与结营）
```

- Day 4 重点：**会评估、会对比、会轻量调参、会看残差**
- Day 4 默认学生已经会：`X/y`、`train_test_split`、训 RF、读 `feature_importances_`、测试 R²
- `07` 是课堂练习册，`08` 是课后练习册，仍然不是新课主体
- **不要**在学生本里写死大纲附录那套「线性回归 0.45 / 森林 0.87」——以本机实测为准
- 本课模拟数据**线性成分不弱**，线性回归测试 R² 可能高于 RF；要教「选模型不只看一次分数」

## 总原则

Day 4 的目标是让学生从「会训一个模型」走到「会评价和选模型」：

1. 会读 R² / MAE / RMSE
2. 会在同一划分上对比多模型
3. 知道超参数与参数的区别
4. 会跑小网格 `GridSearchCV` + 三折 CV
5. 会看残差找猜得差的天
6. 能说清项目默认随机森林的理由

如果课堂节奏变慢，优先保证：

- 三个指标会读
- 对比表至少含 线性 / 浅树 / RF
- 一次小网格搜索
- 一张残差图或 worst 表

XGBoost 安装、大网格、严格公式推导可以砍。

## 本机实测参考量级（random_state=42, test_size=0.2）

> 给学生说「以你跑出来的为准」；下表只供老师心里有数。

| 模型 | 测试 R² 大约 | 备注 |
| ---- | ---- | ---- |
| 线性回归 | ~0.93 | 这份数据很吃线性 |
| 决策树 depth=3 | ~0.52 | 欠拟合感 |
| 决策树 depth=8 | ~0.71 | 训练很高 → 过拟合苗头 |
| 随机森林 100/8 | ~0.76 | 项目默认 |
| KNN k=5 | ~0.16 | 演示「不适合就很差」 |
| GridSearch 小网格 | 常落在 depth=8 附近 | 与固定参数接近也正常 |

## 最低合格线

学生当天至少能独立完成：

1. 在测试集上打印 R² 和 MAE
2. 对比至少 3 个模型的测试 R²
3. 说出 1 个超参数例子
4. 跑通一小次 `GridSearchCV` 并读 `best_params_`
5. 说出残差是什么，并指出误差较大的几天
