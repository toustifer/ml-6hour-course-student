## 正确主线顺序（非常重要）

```
模块 0 环境安装（Python / conda / VS Code / Jupyter / 包）
   ↓
模块 1 概念（不写代码）
   ↓
模块 2 Python 零基础（第一次正式写代码）
   ↓
模块 3 读 CSV
   ↓
模块 4 看懂表
   ↓
模块 5 统计 + groupby
   ↓
模块 6 画图复盘
```

- `00`：单独讲安装与工具，**装不通就不要硬开主课**
- `00` 支持两条路线，**课堂只选一条，不要混装**：
  - 路线 A：官方 Python + pip
  - 路线 B：**conda（Miniconda/Anaconda）+ 环境 `day1_ml`**（机房统一装时推荐）
- `01`：只讲数据是什么，**禁止提前写 pandas**
- `02`：才是 Python 基础课
- `07`：课堂练习册，**不是新课**，是学完对应模块后的巩固

不要出现这种错序：

- 模块 1 就开始 `pd.read_csv`
- 或者把“Python 基础”误当成练习册 07 才开始讲
- 环境还没过，直接进模块 2
- 学生一半 pip、一半 conda 且内核乱选（统一路线后仍要检查内核）

## 总原则

学生默认是**零基础高中生**。

Day 1 的重心不是“尽快进入数据分析”，而是：

1. 先建立“什么是数据表”的感觉（模块 1，不写代码）
2. **把 Python 基础讲透、练够**（模块 2）
3. 再进入 CSV、统计、分组、画图（模块 3-6）

如果课堂节奏变慢，优先保证：

- Python 基础能独立写变量、列表、if、for
- 能读 CSV
- 能做平均数
- 能做天气分组

图表美化可以压缩。

## 为什么 Python 要给这么久

零基础学生常见卡点：

- 不知道代码格子怎么运行
- 分不清数字和字符串
- 列表下标从 0 开始不习惯
- if / for 缩进容易错
- 字典的“键值对应”太抽象

这些如果没打牢，后面的 `pandas` 会整段听不懂。

所以 Day 1 宁可：

- Python 多练 20 分钟
- 后面图表少讲一点

也不要反过来。

## 使用建议

- 模块 0 先保证人人能运行 `print` 和 `import pandas`
- 模块 0 **先宣布课堂统一路线**（A 或 B），不要一半 pip 一半 conda
- 若用 **conda**：要求环境名统一为 `day1_ml`；自检时看 `sys.executable` / `CONDA_DEFAULT_ENV`
- 模块 0 常见卡点：没勾 PATH、**内核没选 `day1_ml`**、打开错文件夹、`conda activate` 在普通 PowerShell 失败（改用 Anaconda Prompt）
- 模块 1 全程口述 + 看图 + 看 markdown 小表，不要打开 pandas
- 模块 2 每讲完一个小概念就立刻让学生运行一个小格子
- 模块 2 不要连续讲超过 10 分钟不练
- 模块 2 和模块 5 最容易拉开差距，要多做板书和口头重复
- 如果学生 Python 明显跟不上，当天可以只推进到“读取 CSV + head()”，统计和画图放到补课或第二天开头补

### 模块 0 · conda 教师口令（路线 B）

课堂上只投这四行即可：

```text
conda create -n day1_ml python=3.11 -y
conda activate day1_ml
conda install pandas matplotlib jupyter -y
conda env list
```

然后 VS Code 打开 notebook → 右上角内核选 **`day1_ml`**。
