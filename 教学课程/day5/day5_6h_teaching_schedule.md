# Day 5 授课节奏安排（经营分析 + 结营中心）

这份文件给老师自己看，不放进学生教案内容里。

> **说明：** 本版 Day 5 **中心是咖啡馆经营深度分析与结营**。
> Flask / `system/` / `mini_predict_app.py` 仅作**可选展示**，主课时间不占满。

## 正确主线顺序

```
模块 0 回顾 + 今天目标（模型→经营→结营）
   ↓
模块 1 经营问题清单
   ↓
模块 2 场景深度对比（groupby / 交叉）
   ↓
模块 3 模型帮决策（重要性 + what-if）
   ↓
模块 4 一页经营诊断报告
   ↓
模块 5 结营路演 + 五天收束
   ↓
模块 6 可选工具（网页 / AI 用法）— 有余力
```

## 总原则

1. **商业表达 > 新 API**：店长听得懂的结论优先
2. 技术复用 Day2–4：分组、RF、R²/MAE、重要性
3. 结营必有：一页纸 + 短路演（哪怕只有 1 名学生也走仪式感）
4. 网页系统：老师 5～10 分钟演示即可；学生主线不依赖 Flask
5. 学生本禁止「你们应该已经会写报告」

## 结营流程建议（模块 5）

1. 每人/每组 3～5 分钟路演
2. 老师 1 分钟反馈（证据、建议是否可执行）
3. 五天链条齐读
4. 合影/寄语/后续学习建议（按机构安排）

## 最低合格线

1. 完成至少两类场景对比并写出发现
2. 训出 RF，Top3 重要性会用业务话说
3. 一次 what-if 或等价推演
4. 一页诊断含 3 条建议
5. 完成短路演或完整口述提纲
