# Day 5 授课节奏安排

这份文件给老师自己看，不放进学生教案内容里。

> **说明：** 时长只写在这里；教师口播/答案在本地 `教师资料/day5/`。

## 正确主线顺序

```
模块 0 五天回顾与路线
   ↓
模块 1 Web 概念 + Flask Hello（终端）
   ↓
模块 2 迷你预测应用（对齐 Day3/4 特征）
   ↓
模块 3 完整系统结构（读 app.py）
   ↓
模块 4 启动完整 system 走通功能
   ↓
模块 5 前端轻量对应
   ↓
模块 6 结课整合与扩展
```

- Day 5 重点：**跑通链路、能讲解**，不是手写全部 HTML/ECharts
- 默认学生会：RF fit/predict、R²、重要性；**不默认会 Flask**
- `07` 练习册，`08` 课后；Hello 与迷你应用在**终端**运行

## 总原则

1. 先迷你、后完整（完整前端太长，避免一上来淹死）
2. notebook 不长期 `app.run`（占内核）；服务在终端
3. 端口：Hello/完整 **5000**，迷你 **5001**
4. 压课时：砍前端美化讲解、砍扩展深讲；保 Hello + 迷你 train/predict + 完整走查 + 结课串讲
