# Day 5 授课节奏（证据 + Flask 汇报外壳 + 结营）

这份文件给老师自己看，不放进学生教案正文。

> **定调：**
> 1. 先有经营证据（场景 + 模型）
> 2. 再用 **Flask 迷你网页**当「给老板看的演示」
> 3. 结营路演 = 话术 + 可点击成果
> Flask **不是**从零 Web 课；完整 `system/` 扫一眼即可。

## 主线顺序

```
0 定调：不能只甩数据
1 老板问题清单
2 场景证据
3 模型证据（重要性 + what-if）
4 Flask 是什么 + 启动迷你演示
5 给老板的汇报话术（边点边讲）
6 结营路演 + 五天收束
7/8 练习与提交
```

## 总原则

1. **汇报语言 > 新 API**
2. 网页服务故事：what-if 给老板看，不讲框架细节
3. 上午证据、下午演示与结营
4. 装不上 flask 时：notebook what-if 顶上，不卡结营
5. 学生本禁止「你们应该已经会写汇报」

## 结营流程

1. 每人 3～5 分钟（鼓励 live 网页）
2. 老师反馈：证据、演示是否服务故事、建议可执行
3. 五天齐读 + 寄语

## 最低合格线

1. 2 条带数字场景发现
2. RF + 重要性业务话
3. 迷你网页跑通 **或** notebook what-if 讲清
4. 完整汇报卡
5. 结营口述过关

## 救火

- flask 装不上 → 跳过安装纠缠，用 notebook 演示，结营不延期
- 端口占用 → 换端口或关旧进程
- 学生只秀网页不讲建议 → 拉回模块 5 时间轴
